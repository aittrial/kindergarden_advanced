## Что такое fixtures (фикстуры)?

Fixtures — это готовые данные, которые pytest создаёт перед тестом автоматически.
Определены в файле `tests/conftest.py`.

| Fixture | Что создаёт | Используется в |
|---------|-------------|----------------|
| `test_kindergarten` | Один садик "Тестовый Садик" | test_03, test_04, test_05 |
| `second_kindergarten` | Второй садик "Другой Садик" | тесты изоляции |
| `test_child` | Ребёнок Иван Петров, без взноса | test_03, test_05 |
| `test_child_with_fee` | Ребёнок Мария Сидорова, взнос 3000 | test_04 |
| `db_session` | Прямая сессия к БД | test_02 |

**Важно:** Перед каждым тестом база данных очищается. Тесты не влияют друг на друга.

### Класс TestHashPassword — тесты функции hash_password()

---

#### test_returns_string
**Что проверяет:** Результат хэширования — строка (тип `str`)  
**Входные данные:** пароль `"mypassword123"`  
**Ожидаемый результат:** функция возвращает объект типа `str`  
**Что произойдёт при успехе:** PASSED  
**Что произойдёт при ошибке:** если функция вернула не строку — FAILED

---

#### test_hash_is_not_plaintext
**Что проверяет:** Хэш не совпадает с исходным паролем (пароль зашифрован)  
**Входные данные:** пароль `"secret"`  
**Ожидаемый результат:** `hash_password("secret") != "secret"`  
**Что произойдёт при успехе:** PASSED  
**Что произойдёт при ошибке:** если хэш совпадает с паролем — система хранит пароли открытым текстом, это критическая уязвимость

---

#### test_sha256_length
**Что проверяет:** SHA-256 всегда производит хэш длиной ровно 64 символа  
**Входные данные:** строка `"любой пароль"`  
**Ожидаемый результат:** `len(hashed) == 64`  
**Что произойдёт при успехе:** PASSED  
**Примечание:** длина хэша фиксирована независимо от длины пароля

---

#### test_deterministic
**Что проверяет:** Один и тот же пароль ВСЕГДА даёт одинаковый хэш  
**Входные данные:** пароль `"consistent_password"` хэшируется дважды  
**Ожидаемый результат:** `hash1 == hash2`  
**Что произойдёт при успехе:** PASSED  
**Почему важно:** если хэш каждый раз разный — нельзя проверить пароль при входе

---

#### test_different_passwords_produce_different_hashes
**Что проверяет:** Разные пароли дают разные хэши  
**Входные данные:** `"password1"` и `"password2"`  
**Ожидаемый результат:** `hash1 != hash2`  
**Что произойдёт при ошибке:** если хэши совпадают — невозможно различить пользователей

---

#### test_empty_string
**Что проверяет:** Пустая строка тоже может быть захэширована (граничный случай)  
**Входные данные:** пустая строка `""`  
**Ожидаемый результат:** возвращает строку длиной 64 символа  
**Примечание:** Edge case — граничный случай, часто источник неожиданных ошибок

---

#### test_long_password
**Что проверяет:** Очень длинный пароль хэшируется без ошибок  
**Входные данные:** строка из 1000 символов `"a" * 1000`  
**Ожидаемый результат:** хэш длиной 64 символа (SHA-256 всегда 64)  

---

#### test_unicode_password
**Что проверяет:** Пароли на кириллице и других языках работают  
**Входные данные:** `"пароль на русском языке"`  
**Ожидаемый результат:** строка длиной 64 символа
