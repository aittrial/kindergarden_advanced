## 1. Введение

### 1.1 Цель документа
Настоящий документ описывает функциональные и нефункциональные требования к системе управления детским садом (KMS). Документ предназначен для разработчиков, тестировщиков и заинтересованных сторон проекта.

### 1.2 Область применения
Система KMS предназначена для автоматизации управления одним или несколькими детскими садами: учёт детей, посещаемость, склад продуктов, расходы, оплата родителей, формирование отчётов.

### 1.3 Определения и сокращения
| Термин | Определение |
|--------|-------------|
| KMS | Kindergarten Management System |
| Суперадмин | Администратор всей системы, управляет несколькими садиками |
| Админ | Сотрудник, управляющий одним садиком |
| Садик | Структурная единица системы (детский сад) |
| Дебитор | Ребёнок, родители которого не оплатили или оплатили частично |

## 2. Общее описание системы

### 2.1 Перспектива продукта
KMS — веб-приложение с многопользовательским доступом. Система разворачивается в облаке (Streamlit Cloud) и использует облачную базу данных (PostgreSQL на Neon.tech). Доступ осуществляется через браузер без установки дополнительного ПО.

### 2.2 Функции системы (краткий обзор)
- Многосадиковая (мультитенантная) архитектура
- Управление детьми (группы, данные родителей)
- Журнал посещаемости
- Склад продуктов с контролем минимального остатка
- Учёт расходов по категориям
- Учёт оплаты родителей (полная/частичная)
- Автоматическое выявление должников
- Формирование отчётов и экспорт в Excel
- Двуязычный интерфейс (русский / английский)
- Поддержка валют (ILS / USD / EUR)

### 2.3 Пользователи системы

| Роль | Описание |
|------|----------|
| **Суперадмин** | Создаётся один. Управляет всеми садиками: создание, редактирование, удаление. Назначает администраторов к садикам. Может входить в любой садик. |
| **Админ** | Привязан к одному садику. Видит только данные своего садика. Управляет детьми, посещаемостью, складом, расходами, оплатой. |

### 2.4 Операционное окружение
| Компонент | Технология |
|-----------|-----------|
| Frontend | Streamlit ≥ 1.35.0 |
| Backend | Python 3.10+ |
| ORM | SQLAlchemy ≥ 1.4, < 3.0 |
| База данных | PostgreSQL (Neon.tech) / SQLite (локально) |
| Деплой | Streamlit Cloud |
| Репозиторий | GitHub |
| Экспорт | pandas + xlsxwriter |

## 3. Функциональные требования

### 3.1 Модуль аутентификации

| ID | Требование |
|----|-----------|
| AUTH-01 | Система должна поддерживать вход по email и паролю |
| AUTH-02 | Пароли хранятся в виде хэша (bcrypt) |
| AUTH-03 | При отсутствии суперадмина отображается вкладка регистрации |
| AUTH-04 | После создания суперадмина регистрация новых пользователей закрыта |
| AUTH-05 | Пользователь может выйти из системы в любой момент |
| AUTH-06 | Язык и валюта сохраняются в профиле и восстанавливаются при следующем входе |

### 3.2 Управление садиками (только суперадмин)

| ID | Требование |
|----|-----------|
| KG-01 | Суперадмин может создать новый садик (название, адрес, телефон, логотип) |
| KG-02 | Суперадмин может редактировать данные садика |
| KG-03 | Суперадмин может удалить садик (каскадное удаление всех данных) |
| KG-04 | Суперадмин видит список всех садиков и может войти в любой |
| KG-05 | При входе в садик суперадмин видит все данные этого садика |
| KG-06 | Суперадмин может вернуться к списку садиков из любой страницы |

### 3.3 Управление администраторами (только суперадмин)

| ID | Требование |
|----|-----------|
| ADM-01 | Суперадмин может создать нового администратора |
| ADM-02 | При создании администратора указывается email, пароль и привязка к садику |
| ADM-03 | Суперадмин может изменить пароль администратора |
| ADM-04 | Суперадмин может удалить администратора |
| ADM-05 | Администратор видит только данные своего садика |

### 3.4 Управление детьми

| ID | Требование |
|----|-----------|
| CH-01 | Система должна хранить: имя, фамилию, дату рождения, имя родителя, телефон родителя, дату зачисления, группу, ежемесячный взнос, статус |
| CH-02 | Группы: младшая, средняя, старшая |
| CH-03 | Статусы: активный, неактивный |
| CH-04 | Возможность фильтрации по группе и статусу |
| CH-05 | Возможность добавления, редактирования и деактивации ребёнка |
| CH-06 | Отображение возраста ребёнка в годах и месяцах |
| CH-07 | Данные изолированы по садику — видны только дети текущего садика |

### 3.5 Журнал посещаемости

| ID | Требование |
|----|-----------|
| ATT-01 | Ежедневная отметка посещаемости для каждого ребёнка |
| ATT-02 | Статусы: присутствовал, отсутствовал, болел |
| ATT-03 | Фильтрация по группе и дате |
| ATT-04 | Отображение итогов по дню (количество присутствующих/отсутствующих) |
| ATT-05 | Нельзя добавить повторную запись за один день для одного ребёнка |

### 3.6 Склад продуктов

| ID | Требование |
|----|-----------|

| PRD-03 | Текущий остаток рассчитывается автоматически (приход − расход) |
| PRD-04 | При остатке ниже минимума — предупреждение на главной странице |
| PRD-05 | История транзакций по каждому продукту |
| PRD-06 | Продукты изолированы по садику |

### 3.7 Учёт расходов

## 4. Нефункциональные требования

### 4.1 Безопасность
- Пароли хранятся исключительно в виде bcrypt-хэша
- Каждый пользователь видит только данные своего садика
- Суперадминская функциональность недоступна обычным администраторам
- Соединение с БД через SSL (Neon.tech)

### 4.2 Производительность
- Страница должна загружаться не более чем за 3 секунды при стандартной нагрузке
- Экспорт Excel — не более 10 секунд для отчётов за месяц

### 4.3 Надёжность
- Приложение использует try/finally для закрытия сессий БД
- При недоступности облачной БД — откат на SQLite (только для локального запуска)

### 4.4 Масштабируемость
- Архитектура поддерживает неограниченное количество садиков
- Каждый садик полностью изолирован по kindergarten_id

### 4.5 Доступность
- Веб-интерфейс, доступен с любого устройства через браузер
- Мобильный браузер поддерживается (responsive layout Streamlit)

## 5. Архитектура системы

### 5.1 Структура файлов
```
kindergarden_advanced/
├── main.py                 # Точка входа, роутинг
├── database.py             # Подключение к БД, engine, Base
├── models.py               # SQLAlchemy модели
├── crud.py                 # CRUD операции
├── auth.py                 # Хэширование паролей
├── auth_guard.py           # Защита страниц, сайдбар
├── i18n.py                 # Переводы, валюты, справочники
├── seed.py                 # Заполнение тестовыми данными
├── requirements.txt
└── pages/
    ├── kindergartens.py    # Управление садиками
    ├── 1_Дети.py           # Управление детьми
    ├── 2_Посещаемость.py   # Журнал посещаемости
    ├── 3_Продукты.py       # Склад
    ├── 4_Расходы.py        # Расходы
    ├── 5_Отчеты.py         # Отчёты
    ├── 6_Админы.py         # Управление администраторами
    └── 7_Оплата.py         # Оплата
```

### 5.2 Модель данных (ERD)

```
Kindergarten (садик)
  └─< User (пользователи/администраторы)
  └─< Child (дети)
        └─< Attendance (посещаемость)
        └─< Payment (платежи)
  └─< Product (продукты)
        └─< ProductTransaction (транзакции склада)
  └─< Expense (расходы)
```

### 5.3 Основные таблицы БД

**kindergartens:** id, name, address, phone, logo_url

**users:** id, email, password_hash, role, language, currency, kindergarten_id(FK)

**children:** id, kindergarten_id(FK), first_name, last_name, birth_date, parent_name, parent_phone, enrollment_date, status, group, monthly_fee

**attendance:** id, child_id(FK), date, status

**products:** id, kindergarten_id(FK), name, unit, min_stock

**product_transactions:** id, product_id(FK), date, quantity, transaction_type

**expenses:** id, kindergarten_id(FK), date, category, amount, description, comment

**payments:** id, child_id(FK), year, month, amount, paid_date, comment

## 6. Внешние интерфейсы

### 6.1 Пользовательский интерфейс
- Streamlit web UI, адаптированный для десктопа и мобильных браузеров
- Навигация через боковое меню (st.navigation)
- Язык/валюта в сайдбаре и на странице входа

### 6.2 База данных
- Production: PostgreSQL на Neon.tech, подключение через DATABASE_URL в секретах Streamlit Cloud
- Development: SQLite файл `local_storage.db`

### 6.3 Экспорт данных
- Формат: Microsoft Excel (.xlsx)
- Библиотека: pandas + xlsxwriter

## 7. Ограничения и допущения

- Система рассчитана на небольшое количество одновременных пользователей (до 20)
- Все даты хранятся в UTC
- Финансовые расчёты производятся в числах с плавающей точкой (float), округление до 2 знаков
- Логотипы садиков хранятся как URL (не как файлы)
- Отсутствует функционал уведомлений по email/SMS
